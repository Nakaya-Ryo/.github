# PharmaForest Governance & Operating Model

## 1. Purpose / Mission

- **Purpose:**
  - To unlock and amplify SAS knowledge that has traditionally remained within individuals, and transform it into shared, collective value.

- **Mission:**
  - To promote open sharing of SAS know-how through SAS Packages, enabling the accumulation of collective knowledge, improving productivity, ensuring quality through standardization, and fostering a vibrant community.

---

## 2. Organization Structure

#### Administrator
- Responsible for **overall initiative/repository governance**
- Define **strategy and roadmap**
- Manage access and permissions (add/remove members)
- Holds final decision-making authority

#### Project Member
- Responsible for **development and maintenance of packages/projects**
- Create and manage issues and pull requests
- Participate in code reviews

---

## 3. Meetings

### Admin Meeting
- Frequency: Monthly
- Attendees: Admin members
- Objectives:
  - Progress updates
  - Discussion of future considerations and issues
  - Technical discussions and knowledge sharing
  - Problem-solving
- Notes:
  - Meeting minutes to be stored in MS Teams
  - Decisions are shared to project members

### PharmaForest Meeting
- Frequency: Bi-monthly
- Attendees: Admin members & Project members
- Objectives:
  - Progress updates
  - Communications and announcements
  - Problem identification and issue extraction
  - Technical discussions and knowledge sharing
- Notes:
  - Meeting minutes to be stored in MS Teams
  - Ad-hoc meetings can be held as needed
---

## 4. Package Policy

### Principles

Based on two key objectives - expanding open source culture among SAS users and enhancing pharmaceutical package development - this repository will include both:

- **General-purpose packages**: To foster broader SAS open source adoption and contribute to the wider SAS community
- **Pharmaceutical-specific packages**: To address unique needs in pharmaceutical data analysis, regulatory compliance, and clinical development workflows

This dual approach ensures we contribute to the overall advancement of SAS open source culture while simultaneously building specialized tools that enhance productivity and quality in pharmaceutical applications.

### Package Types

The repository contains two types of packages:

1. **PharmaForest-created packages**: Packages developed internally within the PharmaForest initiative
   - Developed by our community members
   - Follow our governance and quality standards
   - Maintained directly by the PharmaForest team

2. **Mirrored packages**: Packages from external repositories that are mirrored to PharmaForest
   - Sourced from trusted external repositories
   - Maintained synchronization with upstream sources
   - Provide reliable access to valuable external contributions

### PharmaForest-Created Package Development Process

#### Creation
- Follow community-driven development approach
- Adhere to established design principles and coding standards
- Implement clear documentation

#### Review
- Require peer review from at least one team member
- Critical or core packages would require multiple reviewer approvals
- Review focuses on code quality, documentation, and adherence to standards

#### Testing
- **Core/Critical packages** should have testing process
- Validate with representative datasets when possible
- Document test results when possible

#### Maintenance
- Ongoing maintenance responsibility assigned to original developers and team
- Regular updates to address bugs, security issues, and compatibility
- Version management following semantic versioning principles

#### Licensing
- Use permissive licenses by default (e.g., MIT, Apache 2.0)
